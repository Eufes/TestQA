Детальное описание:
	В документе "Заказ" в табличной части "Товары" после добавления номенклатуры с типом "Услуга" ячейка в колонке "Количество" недоступна для редактирования. При этом если добавить такую номенклатуру, в поле итогового количества по документу она будет учитываться как 1 шт. Например, если в Заказе есть "Номенклатура_Товар" с количеством 5, в данный Заказ добавляется "Номенклатура_Услуга" с количеством 1, то в итоговое количество ТОВАРОВ по документу равно "6" (в должно быть "5" ). Так же изменить количество "Номенклатуры_Услуга" в Табличной Части "Товары" невозможно, т.к. реквизит "Количество" не доступен для редактирования в строке "Номенклатура_Услуга", что является ошибкой.

Шаги для воспроизведения:

	Создать Заказ Покупателя или открыть существующий.
	Добавить в табличную часть "Товары" номенклатуру с типом "товар" в количестве 3 шт.
	Добавить в табличную часть "Товары" номенклатуру с типом "услуга".
	Попробовать изменить количество в последней добавленной строке - будет недоступно для редактирования.
	Проверяем общее количество в поле "Количество (итог)" - будет 4.

Фактический результат: 
	После добавления номенклатуры с типом "Услуга" в строку табличную часть "Товары" - ячейка в колонке "Количество" недоступна для редактирования. При этом итоговое количество "Товаров" (поле "Количество (итог)") включает и количество по номенкллатурам с типом "Услуга".

Ожидаемый результат:
	После добавления номенклатуры с типом "Услуга" в строку табличную часть "Товары" - ячейка в колонке "Количество" доступна для редактирования. Итоговое количество "Товаров" (поле "Количество (итог)") по прежнему включает и количество по номенкллатурам с типом "Услуга".
